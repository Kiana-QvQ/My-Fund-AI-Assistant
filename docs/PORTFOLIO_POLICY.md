# 保守进阶型指数定投组合 README

## 一、方案概述
本组合面向**风险偏好保守、追求长期稳健增值**的投资者，以「A 股宽基为核心、美股小仓位分散、短债资产做安全垫」为核心架构，通过定投方式平滑成本，配合估值分位规则优化买入节奏，在控制最大回撤的前提下获取市场长期平均收益。

组合以跑赢居民消费价格指数（CPI）通胀为基础保值目标，长期力争向广义货币供应量（M2）增速靠拢，在低波动前提下实现资产稳健增值。近5年国内CPI年均通胀率约1.8%，M2年均增速约8.2%，本组合长期收益可稳定覆盖通胀，权益行情较好阶段可接近M2增速水平。

- **组合定位**：稳健增值型，权益与固收均衡配置
- **建议持有周期**：3 年以上
- **预期年化收益**：长期 3.2%~7.6%（随市场周期波动）
- **历史最大回撤参考**：约 -18%（高点入场 5 年维度）

---

## 二、核心投资理念
1. **A 股为主，美股为辅**：沪深 300 + 中证 500 构成权益主力，押注国内经济长期增长；标普 500 + 纳斯达克 100 小仓位配置，分散单一市场风险，赚取全球龙头企业盈利增长收益。
2. **固收打底，控制波动**：短债类资产作为组合压舱石，大幅降低账户整体回撤，提升持有体验，避免因短期波动中断定投。
3. **低估买入，高估暂停**：以指数估值历史分位为锚，只在偏低估区间定投加仓，中等估值以上停止新增投入，避免高位无脑接盘。
4. **长期持有，定期再平衡**：不博弈短期行情，依靠企业盈利增长和估值周期赚钱；每年校准一次仓位比例，维持风险收益结构稳定。

## 五、场内 vs 场外：区别与费用全拆解
### 1. 基础定义
- **场内**：在证券交易所挂牌交易，需开通证券账户，像股票一样实时买卖，价格全天波动。ETF 均为场内品种。
- **场外**：在交易所外的基金销售渠道申购赎回，全天只有一个收盘净值，按金额申购、按份额赎回。ETF 联接基金属于场外品种。

### 2. 核心区别对比
|对比维度|场内 ETF|场外联接基金（A 类）|
|---|---|---|
|交易渠道|证券账户|第三方基金平台、银行、基金公司官网|
|成交价格|全天实时报价，每秒波动|每日 1 个收盘净值，15:00 前下单按当日净值|
|最低买入门槛|1 手 = 100 份，必须 100 份整数倍|通常 10 元起，可精确到分|
|到账速度|卖出资金当日可用，次日可取|赎回到账 T+1~T+4，QDII 为 T+7+|
|自动定投|需通过条件单实现，受门槛限制|原生支持自动扣款定投，比例精准|

### 3. 费用完整明细
#### 共同费用：年运作费（管理费 + 托管费）
所有渠道一致，每日从基金净值中自动扣除，投资者无需额外支付，看到的净值已扣除此项费用。

#### 场内交易费用
|费用项|收费规则说明|
|---|---|
|交易佣金|按成交金额比例收取，普遍万 0.5~ 万 1；多数券商默认**单笔最低 5 元**（买卖双向收取）；部分券商针对场内基金可申请取消最低限制|
|过户费|沪市品种收取十万分之 0.2，双向，金额极低可忽略|
|印花税|ETF 免征|
|隐性成本|1. 100 份门槛导致小额资金闲置；2. QDII 品种存在溢价损耗（市价高于净值）；3. 买卖滑点|

#### 场外交易费用（A 类份额，长期持有首选）
|费用项|收费规则说明|
|---|---|
|申购费|原费率 1.2%，第三方平台普遍 1 折，实际约 **0.12%**，买入时一次性扣除|
|赎回费|持有时间越长越低；持有 ≥ 2 年通常全额免除；持有不满 7 天收取 1.5% 惩罚性费用|
|销售服务费|A 类无，C 类按年计提（短期持有选 C 类）|
|隐性成本|几乎无溢价损耗；QDII 存在少量换汇摩擦成本，已体现在净值中|

### 4. 成本适用场景结论
- **每月定投金额 ≤ 500 元**：场外方案综合成本更低，无门槛限制，可精准执行比例，推荐场外
- **每月定投金额 ≥ 2000 元**：单笔金额覆盖最低佣金门槛后，场内 ETF 综合成本更低，推荐场内
- **大额一次性买入**：场内 ETF 费率优势明显，优先选择场内

## 六、定投执行策略
### 1. 初始建仓规则
- 初始本金建议分 **6 个月分批建仓**，平滑入场成本，避免一次性买在高位
- 第 1 个月投入总建仓资金的 20%，第 2~6 个月每月投入剩余资金的 16%
- 资金严格按 51% / 27% / 11% / 8% / 3% 的目标仓位比例分配至各标的

以初始本金 10000 元为例：
|建仓阶段|当月总投入|短债底仓（51%）|沪深300（27%）|中证500（11%）|标普500（8%）|纳指100（3%）|
|---|---|---|---|---|---|---|
|第1个月|2000元|1020元|540元|220元|160元|60元|
|第2~6个月（每月）|1600元|816元|432元|176元|128元|48元|

### 2. 每月持续定投
- 建仓完成后，按固定金额每月定投，建议金额 300~500 元起步，可随收入提升逐步增加
- 每月资金按 51% / 27% / 11% / 8% / 3% 的目标比例拆分

以每月定投 300 元（场外方案）为例：
|标的|占比|每月定投金额|实操说明|
|---|---|---|---|
|嘉实超短债债券 A（012773）|51%|153元|A 类份额、无销售服务费，适合长期持有|
|华泰柏瑞沪深300ETF联接 A（460300）|27%|81元|对应场内 510300，支持场外定投|
|南方中证500ETF联接 (LOF) A（160119）|11%|33元|全行业宽基中证500，LOF属性不影响场外定投|
|博时标普500ETF联接 (QDII) A（050025）|8%|24元|跟踪标普500全市场宽基，注意QDII溢价和额度|
|南方纳斯达克100指数发起（QDII）A（016452）|3%|9元|额度相对稳定，适合小额定投；最低起投额以平台实际规则为准|

### 3. 场内「蓄水池」执行方案（解决 100 份门槛问题）
由于场内 ETF 最低 100 份买入，小额定投无法直接拆分至每只标的，采用以下机制：
1. 每月定投资金全部买入短融 ETF（或证券账户闲置理财），作为资金蓄水池
2. 当对应指数估值达标、且蓄水池资金足够买入 1 手对应 ETF 时，再调拨资金买入
3. 长期维持 51/27/11/8/3 的目标仓位比例，每年做一次再平衡校准

## 七、买入与估值执行策略
本策略以指数PE-TTM历史分位为核心锚点，围绕40%分位划分操作区间，严格执行“低估加仓、高估暂停、分批补投”的纪律，避免高位追涨、低位踏空。

### A股指数（沪深 300、中证 500）
#### PE 历史分位 ＜ 40%（低估/合理偏低区间）
- 30% ＜ 分位 ＜ 40%：执行**正常定额定投**，按目标比例分配当月资金
- 分位 ≤ 30%：执行**双倍定额定投**，将当月对应权益类定投金额翻倍，加速收集低价筹码；翻倍部分资金从短债底仓调拨
- 操作频率：每月固定时点执行1次，不盘中追涨杀跌

#### PE 历史分位 ≥ 40%（合理偏高/高估区间）
- 40% ≤ 分位 ＜ 60%：**暂停该指数的新增定投**，当月对应计划投入的资金全部转入短债底仓，积攒备用资金
- 分位 ≥ 60%：执行**分批止盈**，逐步卖出该指数1/3~1/2的仓位，止盈所得资金全部转入短债底仓，锁定收益
- 补投规则：当指数估值回落至40%分位以下后，用前期积攒的短债备用金，分3~6个月分批补投对应指数，逐步回归目标仓位比例

### 美股指数（标普 500、纳斯达克 100）
参考自身10年PE历史分位，阈值适当放宽，操作逻辑与A股一致：
- 分位 ＜ 50%：正常定投
- 50% ≤ 分位 ＜ 70%：暂停新增定投，资金转入短债底仓
- 分位 ≥ 70%：分批止盈
- 额外硬性约束：场内 QDII-ETF 溢价率 **＞ 2%** 时，无论估值高低均暂缓买入，规避溢价损耗；溢价回落至1%以内再分批进场

## 八、再平衡规则
1. **再平衡周期**：每年 1 次，固定在年末或年中执行
2. **触发条件**：任意一类资产偏离目标仓位比例 ±5% 以上
3. **操作方式**：卖出超配部分，买入低配部分，回归目标仓位；定投阶段也可通过调整每月定投分配比例，逐步回归目标，减少交易摩擦

---

## 九、风险提示
1. **市场波动风险**：权益类资产短期波动较大，熊市阶段可能出现 18% 左右账面浮亏，属正常现象，需做好资金规划，用长期不用的闲钱投资
2. **汇率风险**：美股 QDII 资产以美元计价，人民币大幅升值会侵蚀人民币口径收益；小仓位配置下对整体组合影响有限
3. **QDII 额度风险**：外汇额度紧张时，场外 QDII 基金可能限购，场内 QDII-ETF 可能出现高溢价
4. **信用风险**：短债资产底层为高评级债券，违约概率极低，但非刚性兑付，极端市场环境下可能出现短暂小幅回撤
5. **历史业绩不代表未来**：所有回测数据仅为历史模拟，不构成收益承诺

## 十一、AI 规划与交易日记项目结构

本项目采用“上游项目只读、本项目编排层独立”的方式接入 AI，避免修改或破坏
GitHub 拉取的第三方项目。

```text
integrations/
  upstream/                 第三方项目的只读副本
    ai-berkshire/
    vibe-trading/
  adapters/                 我们自己的适配器，不放入上游目录
app/
  ai_assistant.py           规划、日记和复盘入口
config/
  portfolio_policy.json     可供程序读取的投资政策
journal/                    个人交易日记
reports/                    AI 生成的报告
data/                       行情、估值和账户导出数据
logs/                       本地运行日志
docs/AI_INTEGRATION.md      AI 接入边界和分阶段方案
```

### AI 的职责边界

- `plan`：根据投资政策和最新观察生成定投/再平衡建议
- `journal`：把交易事实、判断、情绪和结果整理成日记草稿
- `review`：检查交易是否偏离 README 中的仓位和估值规则
- 默认只提供建议和草稿，不自动下单，必须人工确认

### 本地运行

先配置 `config/.env.example` 中的 AI 服务环境变量，然后运行：

```powershell
python app/ai_assistant.py --mode plan --input "本月可投入300元，510300估值分位32%，当前短债53%"
python app/ai_assistant.py --mode journal --input journal/daily/2026-07-16-template.md
python app/ai_assistant.py --mode review --input reports/latest-trade.md
```

第三方项目的更新、依赖安装和运行都应在各自的
`integrations/upstream/<项目名>` 目录内进行；不要直接修改这些目录中的文件。
